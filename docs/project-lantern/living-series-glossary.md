# Living Series Glossary

## Project Lantern

Project Lantern is personalized episodic entertainment.

It is a product for shaping and returning to ongoing personalized story experiences. It should feel closer to a streaming app than a writing tool.

## Living Series

A Living Series is the core object in Project Lantern.

It is an ongoing personalized series with cast, world, tone, episodes, saved moments, favorites, and continuation. A Living Series is meant to be shaped, watched, saved, continued, and returned to later.

## Story Shaper

A story shaper is the user role Project Lantern is designed around.

Users guide the experience through preferences, choices, personalization, favorites, and continuation. They are not expected to act as authors, manage story structure, or write full scenes.

## Episode

An Episode is an installment inside a Living Series.

It should feel like part of an ongoing entertainment experience, not a standalone text generation.

## Cast

Cast means the people, roles, relationships, and recurring characters that make a Living Series feel personal and recognizable.

## World

World means the setting, rules, mood, places, history, and recurring context that help a Living Series feel coherent over time.

## Make This Mine

Make This Mine is the primary personalization action.

It turns a story idea, series, cast, world, or episode direction toward the user's own preferences. It should be favored over workflows that ask users to write or edit like authors.

## Save

Save keeps a Living Series, episode, or meaningful story state available for later.

Save supports the return path in the core loop.

## Favorite

Favorite marks a Living Series, episode, character, world detail, or moment as especially meaningful to the user.

Favorites should help the product understand what the user wants more of.

## Continue Series

Continue Series is the path for creating the next episode or next meaningful installment in an existing Living Series.

It should be visible, inviting, and central to the return experience.

## Return Later

Return Later is the expectation that users can leave Project Lantern and come back to a Living Series without losing the thread.

The product should preserve enough context that resuming feels natural.

## Core Loop

The Project Lantern core loop is:

Create Living Series -> Add Cast/World -> Create Episode -> Save/Favorite -> Continue Series -> Return Later.

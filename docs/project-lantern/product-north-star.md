# Project Lantern Product North Star

## Definition

Project Lantern is personalized episodic entertainment.

It helps people shape ongoing, streaming-style story experiences that remember what they love, who they care about, and where they want the story to go next. The product should feel like opening a favorite series and choosing the next episode, not like sitting down to operate a writing tool.

## Core Object

The core object is a Living Series.

A Living Series is an ongoing personalized entertainment experience with its own cast, world, tone, memory, episodes, favorites, and continuation path. It is meant to be returned to over time.

## User Role

Users are story shapers, not authors.

They should be able to guide taste, tone, cast, world details, episode direction, and personal meaning without needing to write scenes, manage outlines, or think like professional writers.

## Core Loop

Create Living Series -> Add Cast/World -> Create Episode -> Save/Favorite -> Continue Series -> Return Later.

Every major product decision should support this loop clearly and reduce friction between those steps.

## Experience Principles

- Project Lantern should feel like a streaming app, not a writing tool.
- Living Series should be discoverable, resumable, and emotionally specific.
- Episodes should feel like installments inside an ongoing show, not isolated generations.
- The product should make returning later feel natural and rewarding.
- Creation controls should be simple, inviting, and focused on shaping the experience.

## Primary Personalization Action

Make This Mine is the primary personalization action.

It should represent the moment when a user takes a story, premise, cast, world, or episode direction and turns it toward their own tastes. When in doubt, personalization flows should converge on Make This Mine instead of exposing raw authoring complexity.

## Product Guardrails

- Do not turn Project Lantern into a blank-page writing app.
- Do not make users responsible for professional story structure.
- Do not over-index on prompts as the main interface.
- Do not treat episodes as disposable one-offs when they belong to a Living Series.
- Do not hide continuation, saving, favorites, or return paths behind secondary workflows.
